Multi-threading, WebAssembly-driven and SIMD-optimized Audio Engine Experiment

This simple experiment demonstrates how to use WebAssembly to create a multithreading, SIMD-optimized audio engine using Rust. The architecture is quite unique, as the Web Audio API is only used for one AudioWorkletProcessor that receives the audio data from the WebAssembly-based Audio Engine via an atomic, lock-free, wait-free, and zero-copy RingBuffer that wraps a SharedArrayBuffer that is shared between the WebAssembly, the AudioWorklet thread, and the main thread.
